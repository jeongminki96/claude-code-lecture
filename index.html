SECTION 01

LLM이란 무엇인가

LLM(거대 언어 모델) 은 “방대한 텍스트로 학습해, 다음에 올 토큰(단어 조각)을 확률적으로 예측하는 방식으로 인간 언어를 이해하고 생성하는 AI 모델”입니다.

🍞 쉬운 비유 — “엄청난 양의 글을 읽고, 다음에 올 말을 가장 그럴듯하게 맞히는 거대한 언어 모델”

🎯

① 다음 토큰 예측

문장에서 다음에 올 단어 조각을 확률로 골라 이어 붙입니다.

🌐

② 거대한 규모

수천억 개의 매개변수와 인터넷 규모의 데이터로 학습합니다.

🛠️

③ 사전학습 + 미세조정

먼저 폭넓게 배우고, 이후 용도에 맞게 다듬습니다.

📖

④ 문맥 이해

한 번에 기억하는 분량(컨텍스트 윈도우) 안에서 앞뒤를 봅니다.

✨

⑤ 창발적 능력

규모가 커지면 가르치지 않은 능력이 갑자기 나타납니다.

⚠️

⑥ 환각

그럴듯하지만 틀린 내용을 지어낼 수 있어 확인이 필요합니다.

🔮 직접 해보기 — “다음 단어 맞히기”

문장을 입력하는 대로(또는 버튼을 누르면) LLM이 다음에 올 단어(토큰) 후보를 확률(%)로 매기는 모습을 실시간으로 보여줍니다. 시뮬레이션

1
후보마다 확률을 매깁니다. (위 막대그래프 = 모델 머릿속의 점수)
2
이 중 딱 하나를 골라 내보냅니다. 보통 가장 확률 높은 단어를 고르고, 설정(‘온도’)을 높이면 일부러 덜 뻔한 단어를 골라 다양하게도 합니다.
3
고른 단어를 문장 뒤에 붙이고, 다시 1번부터 반복합니다. → 이렇게 한 단어씩 이어 문장이 만들어집니다.
최종 출력(output)

🔓 공개 방식 비교

모델을 어떻게 공개하느냐에 따라 두 갈래로 나뉩니다.

⚖️ 핵심 맞바꿈(trade-off) — 폐쇄형은 성능이 앞서지만 자료를 밖으로 보내야 하고, 개방형은 내부망에서 안전하게 돌릴 수 있는 대신 성능이 다소 낮습니다. 그래서 기관은 “보안이 중요하면 오픈 모델 + 파인튜닝”을 자주 택합니다. (↓ 아래에서 직접 만들어 보세요)

GPT vs ChatGPT

GPT는 ‘엔진’, ChatGPT는 그 엔진을 얹은 ‘완성차’ 입니다.

🌾→🍞

GPT = 밀가루 / ChatGPT = 빵

재료(GPT)만으로는 먹기 어렵고, 빵(ChatGPT)으로 가공해야 누구나 씁니다.

⚙️→🚗

GPT = 전기모터 / ChatGPT = 테슬라

핵심 동력(GPT)에 차체·핸들·화면(ChatGPT)을 얹어 완성품이 됩니다.

🔧 직접 해보기 — 엔진에 기능을 하나씩 얹어 완성차 만들기

버튼을 누를 때마다 GPT(엔진) 에 기능이 하나씩 붙어 ChatGPT(완성차) 가 됩니다.

⚙️ GPT (엔진) — 그 자체로는 ‘다음 단어 예측기’

아래 버튼으로 기능을 얹어 보세요.

💬 채팅 화면(UI) — 사람이 편하게 대화
🧠 대화 기억 — 앞에서 한 말을 이어 감
📎 파일 업로드 — 문서를 읽고 답함
🎙️ 음성 대화 — 말로 묻고 들음

🏗️ 직접 해보기 — 오픈 모델로 ‘재정경제부 전용 AI’ 만들기

폐쇄형 모델은 우리가 손댈 수 없지만, 개방형(오픈) 모델은 내려받아 우리 데이터로 다시 가르칠(파인튜닝) 수 있습니다. 4단계로 직접 만들어 보세요.

1오픈 모델 가져오기
2학습 데이터 고르기
3파인튜닝(추가 학습)
4전용 모델 완성
① 토대가 될 오픈 모델 (이미 똑똑하지만, 우리 기관은 아직 모름)
Llama 3Meta · 오픈
Qwen알리바바 · 오픈
GemmaGoogle · 오픈
② 무엇을 더 가르칠까요? (재료 = 우리 데이터를 골라 보세요)
같은 질문을 던져 봅니다 👉 “예비비를 다른 곳에 쓰려면 어떻게 하나요?”
🔹 파인튜닝 전 (오픈 모델 그대로)
일반적으로 예비비는 예측하지 못한 지출에 대비한 예산이며, 사용하려면 보통 내부 승인 절차가 필요합니다. (두루뭉술 — 우리 기관 규정·용어는 모름)
🔸 파인튜닝 후 (재정경제부 전용)
아직 학습 전입니다. 위 ③ 버튼을 눌러 파인튜닝을 실행해 보세요.

👉 핵심: 오픈 모델 = ‘똑똑한 신입’, 파인튜닝 = ‘우리 업무를 가르치는 교육’. 같은 두뇌라도 무엇을 더 가르치느냐에 따라 ‘우리 기관 전용 AI’가 됩니다.

🔀 그래서 — 모델 ‘하나’가 아니라 등급별 AI 파이프라인

국정원 N2SF(국가 망 보안체계)는 데이터를 C(기밀)·S(민감)·O(공개) 세 등급으로 나눕니다. 그래서 AI도 모델 하나로 다 하지 않고, 업무의 보안등급에 맞는 모델로 보내는 파이프라인으로 짭니다. 아래 업무를 눌러 보세요 — 어디로 보내지는지 보여 줍니다.

🔀 N2SF 등급 분류기 — 데이터 민감도에 따라 경로 결정

💡 핵심: 보안 등급에 따라 모델을 갈아 끼우는 하이브리드 AI 파이프라인 — 기밀(C)은 내부망 오픈모델로 안전하게, 공개(O)는 강력한 클라우드 모델(Claude·GPT)로 빠르게. 한 기관 안에서 여러 모델이 협업합니다. (우리가 배울 Claude Code는 O·S 업무에서 큰 힘을 발휘합니다.)

SECTION 02

IDE란 무엇인가

IDE = Integrated Development Environment (통합 개발 환경) — 코드를 쓰고 · 실행하고 · 고치는 일을 한곳에서 하는 작업 공간입니다.

🗂️ IDE는 ‘여러 도구가 한 화면에’ 모여 있습니다

실제 IDE 한 창에는 파일 트리 · 편집기 · AI 에이전트 · 터미널 이 함께 들어 있습니다. (Claude Code는 이 중 ‘AI 에이전트 + 터미널’ 자리에서 움직입니다)

SECTION 03

CLI란 무엇인가

CLI = Command Line Interface (명령줄 인터페이스) — 마우스 클릭 대신 글자 명령으로 컴퓨터에 직접 일을 시키는 창구입니다.

🧪 직접 해보기 — 가짜 터미널

아래 명령어 버튼을 누르면, 진짜 터미널처럼 결과가 한 글자씩 출력됩니다. (안전한 예시 · 실제 실행 아님)

terminal — 재정경제부
명령어 버튼을 눌러 보세요 👇

⚠️ 그런데 — 지금 AI 도입의 ‘병목’ 중 하나가 바로 이 CLI입니다

요즘 강력한 AI 코딩 도구(예: Claude Code)는 GUI 버튼이 아니라 CLI(터미널) 를 기본 무대로 삼습니다. 성능은 뛰어나지만, 현장에 퍼지지 못하게 막는 두 가지 장벽이 있습니다.

⌨️

① CLI 환경에 대한 낯섦

대부분의 실무자는 마우스·클릭(GUI) 에 익숙합니다. 검은 화면에 명령어를 직접 쳐야 하는 CLI는 처음엔 진입장벽이 높아, “좋다는 건 알겠는데 손이 안 가는” 상태가 됩니다.

🚚

② 배포·설치의 어려움

📏 ‘AI 코딩 실력’은 어떻게 재나요? — SWE-bench

SWE-bench는 Software Engineering(소프트웨어 개발) 실력을 재는 시험이라는 뜻입니다. 위 ③ 코딩능력 80%대의 근거가 바로 이 시험으로, 장난감 문제가 아니라 실제 오픈소스(GitHub)에 올라온 진짜 버그를 AI가 직접 고치게 합니다.

1
실제 GitHub 프로젝트의 진짜 버그·이슈를 줍니다. (실무에서 마주치는 그대로)
2
AI가 코드를 읽고 직접 수정(패치)을 만들어 냅니다.
3
원래 실패하던 테스트가 통과하면 정답으로 자동 채점. (사람 주관 없이 객관적)
4
맞힌 비율이 점수. 80%대 = 실제 이슈 5개 중 4개를 사람 손 없이 해결한다는 뜻.

🗞️ 최근 화제 — Mythos & Fable ‘사태’

2026년 상반기, 클로드는 역대급 성능과 안전·규제 논쟁의 한복판에 서며 화제가 됐습니다. 사실관계를 시간순으로 정리하면 —

2026.4.7
Mythos 공개 + Project Glasswing — Anthropic이 차세대 모델 Mythos Preview를 공개했습니다. 주요 OS·브라우저에서 제로데이 취약점 1,000건 이상을 찾아낼 만큼 강력해, 일반에는 공개하지 않고 방어 목적으로 약 40개 기관에만 제한 제공하는 Project Glasswing(애플·구글·MS·엔비디아·AWS·팔로알토 등)을 함께 출범했습니다. (앞서 3월 26일 내부 문서 유출로 Mythos의 존재가 먼저 알려졌습니다.)
2026.6.초
빠른 확산 — Mythos 접근이 EU와 150여 개 기관(15개국 이상)으로 확대됐습니다.
2026.6.9
Fable 5 일반 공개 — Mythos급 성능을 안전하게 다듬어 처음으로 일반에 공개한 모델입니다. 소프트웨어 개발·지식 업무·과학 등 대부분의 벤치마크에서 최고 수준(SOTA)을 기록했습니다. (정부 인접 사이버보안용 제한판 Mythos 5도 함께 출시) “AI가 너무 위험해지고 있다”는 경고 며칠 뒤의 공개라 더 화제가 됐습니다.
2026.6.12
중단 ‘사태’ — 공개 사흘 만에 美 정부가 국가안보 우려로 수출통제를 지시하자, Anthropic은 Fable 5·Mythos 5를 전 세계에서 중단했습니다. 사유는 Fable 5의 ‘탈옥(jailbreak)’ 보고였는데, 실제로는 보안 방어자들이 흔히 쓰는 코드 분석 기능이었습니다. Anthropic은 “좁은 탈옥 하나로 수억 명이 쓰는 모델을 리콜하는 건 부당하다”며 공개 반박했습니다.

👉 핵심: 강력함(취약점까지 찾아내는 코딩 실력)과 그로 인한 안전·규제 논쟁이 맞물리며, 클로드는 ‘성능’과 ‘화제성’ 모두에서 주목받고 있습니다.

SECTION 06

Claude Code를 쓰는 3가지 방법

Claude Code는 ‘어디서 쓰느냐’에 따라 역할이 달라집니다. 같은 두뇌라도 무대에 따라 — 가볍게 물어보는 도구가 되기도, 프로젝트 전체를 고치는 손이 되기도 합니다.

🌐

웹·클라우드에서 쓰기 (claude.ai/code)

설치 없이 브라우저에서 Claude Code에게 일을 맡기면, 클라우드가 작업하고 결과를 정리해 주는 방식

가장 쉬움
👍 장점

설치가 필요 없음. 브라우저(휴대폰 포함)에서 작업을 맡기면 클라우드가 알아서 처리하고, 여러 작업을 동시에 돌릴 수도 있습니다.

⚠️ 단점

GitHub 저장소에 연결해 변경을 PR(검토 요청)로 받는 방식이라 GitHub·PR 개념이 조금 필요합니다. (현재 Pro/Max/Team 베타)

🙋 추천 대상

설치 없이 가볍게 시작하려는 사람, 입문자, 외부에서·휴대폰으로 맡기고 싶은 사람

🧰

Cursor 같은 IDE 확장으로 쓰기

Cursor·VS Code 같은 개발환경 안에서 확장 기능처럼 Claude를 쓰는 방식 — 사람이 직접 개발할 때 편합니다

비개발자 입문 실습

⚠️ 단점

IDE 기본 사용법은 알아야 하고, CLI 전용 기능 일부는 확장에서 안 됩니다(자동화·헤드리스·훅 등).

🙋 추천 대상

비개발자·바이브코딩 입문자, 실습형 수강생, 직접 만들어보고 싶은 사람

터미널에서 claude 명령으로 실행하는 방식 — 명령어 기반이라 다른 시스템·스크립트가 AI를 불러 쓰기 좋습니다

자동화·연동
👍 장점

⚠️ 단점

터미널·로그인·권한·폴더 위치 개념이 필요해, 사람이 손으로 직접 개발하기엔 IDE보다 덜 편할 수 있습니다.

🙋 추천 대상

자동화·CI/CD·서버 연동에 AI를 끼워 넣으려는 사람, 반복 작업을 스크립트로 돌리려는 실무자·개발자

💡 이렇게 말해보세요

오류를 재현하고 원인을 찾아 고쳐줘
묻지 말고 가능한 범위에서 끝까지 진행해줘

👉 웹(설치 없이 시작) · IDE 확장(사람이 개발할 때 편함) · CLI(자동화 시스템에 끼워 넣기 좋음). 코딩 실력은 셋 다 같습니다 — 어디서·어떻게 쓰느냐(편의 vs 자동화)만 다릅니다.

SECTION 10

하네스 엔지니어링

좋은 결과는 좋은 질문 하나만으로 나오지 않습니다. 모델을 감싼 ‘환경(하네스)’을 잘 설계해야 합니다. — 프롬프트 엔지니어링을 넘어, 하네스 엔지니어링으로.

🦺 하네스(harness)란?

하네스는 LLM을 ‘에이전트’로 만들어 주는 골격입니다. 모델은 혼자선 ‘다음 단어 예측’만 할 뿐 — 도구를 쓰고 · 파일을 읽고 · 명령을 실행하고 · 맥락을 기억하는 일은 모두 모델을 감싼 하네스가 해 줍니다.

🧠

모델 = 두뇌(엔진)

똑똑하지만, 그 자체로는 생각만 합니다. 손도 발도 기억도 없습니다.

🦾

하네스 = 몸체

두뇌에 손·발·기억·작업대를 붙여 ‘일하는 에이전트’로 만듭니다. Claude Code 자체가 Claude를 위한 하네스입니다.

이 ‘일하는 과정’을 에이전트 루프라고 합니다 — 한 번에 끝내지 않고, 보고 → 하고 → 다시 보며 반복합니다.

요청→ 맥락 구성→ 모델 추론→ 도구 실행→ 결과 관찰↺ 완료

🐎 비유로 이해하기 — 야생마에 고삐 얹기

좋은 모델은 야생마와 같습니다 — 힘세고 영리하지만 통제되지 않아 어디로 튈지 모릅니다. 여기에 고삐·안장·편자·마구 같은 장비(=하네스)를 하나씩 얹으면, 점점 ‘일하는 말’로 길들여집니다. 아래 ‘장비 얹기 ▶’를 눌러, 각 장비가 어떤 AI 하네스 요소에 해당하는지 따라가 보세요.

✨ ✨ 🏅
🐎

길들임(통제력) 0%
🎯고삐 🪑안장 🧩편자 🔌마구 🤠기수
🐎 야생마 — 길들이기 전 모델(LLM)
힘세고 영리하지만, 그 자체로는 제멋대로입니다. 잠재력은 100점 — 하지만 방향도, 집중도, 멈출 줄도 모릅니다. ‘장비 얹기 ▶’를 눌러 길들여 보세요.

서브에이전트

큰 일을 쪼개 전문 에이전트 여럿이 나눠·병렬로 처리하게 합니다.

특정 시점에 자동으로 동작 — 검증·차단·알림·로깅을 끼워 넣습니다.

🧠

메모리

🛡️

권한·안전장치

위험하거나 되돌리기 어려운 작업은 확인을 받거나 막습니다.

🔌 MCP — 외부 세계와 잇는 ‘표준 콘센트’

MCP(Model Context Protocol)는 모델을 외부 도구·시스템에 연결하는 표준 규격입니다. 모델 혼자선 닿지 못하는 메일·드라이브·캘린더·사내 DB·브라우저 같은 바깥 세계를, MCP를 통해 도구처럼 쥐여 줍니다. 앞의 비유에서 ‘마구(연결 장치)’에 해당하는 부분이죠.

모델↔ MCP (표준 규격)↔ 외부 도구·데이터
🔌

표준 규격(콘센트)

누가 만들든 같은 규격이면 그대로 꽂힙니다 — USB처럼. 도구마다 따로 붙일 필요가 없습니다.

🌐

바깥 세계 연결

메일·드라이브·캘린더·사내 시스템 등 실제 업무 도구를 모델이 직접 읽고 쓰게 합니다.

🧰

도구를 ‘확장’

하네스의 도구 계층을 바깥으로 넓히는 부분 — 필요할 때 꽂고, 필요 없으면 뺍니다.

👉 예: ‘사내 회계 시스템’ MCP 서버를 연결하면 — 모델이 직접 집행 내역을 조회해 보고서에 반영합니다.

🧩 스킬(Skills) — 필요할 때 꺼내 쓰는 전문 능력

스킬은 ‘하나의 능력’을 폴더에 담아 둔 것입니다. 사용법(SKILL.md)과 스크립트·참고자료를 모아 두면, 모델이 관련 상황에서 알아서 꺼내 씁니다.

my-skill/ SKILL.md # 무엇을 · 언제 · 어떻게 (설명서) scripts/ # 실행할 스크립트 references/ # 참고 문서 · 양식 · 템플릿

👉 예: ‘재정 보고서 양식’ 스킬 — 보고서를 쓸 때만 자동으로 우리 기관 양식·문체를 불러옵니다.

📦 플러그인(Plugins) — 능력들을 묶은 ‘설치 꾸러미’

플러그인은 스킬 · 슬래시 명령어 · 서브에이전트 · 훅 · MCP 서버를 하나의 설치 가능한 꾸러미로 묶은 것입니다. 스킬이 ‘앱 하나’라면, 플러그인은 ‘여러 앱을 담은 패키지’예요.

스킬+ 명령어+ 서브에이전트+ 훅+ MCP= 플러그인

앱스토어처럼 마켓플레이스에서 받아 한 줄로 설치합니다 — 팀 전체가 같은 명령어·같은 규칙·같은 도구를 한 번에 갖춰 ‘표준화’됩니다.

/plugin marketplace add our-org/claude-plugins # 마켓플레이스 등록 /plugin install finance-toolkit # 꾸러미 설치 → 끝

⚖️ 스킬 vs 플러그인 — 한눈에

🧩

스킬

단위 능력 하나
· 모델이 자동으로 꺼내 씀
· 폴더 + SKILL.md
· 가볍게 ‘기능’을 추가

📦

플러그인

여러 능력의 꾸러미
· 사람이 설치해 사용
· 스킬·명령어·훅·MCP 묶음
· 마켓플레이스로 배포·공유

💡 정리 — 하네스는 ‘일하는 환경’, 스킬은 그 환경에 더하는 ‘능력 하나’, 플러그인은 그 능력들을 묶어 ‘팀에 배포하는 패키지’. 세 가지를 잘 다루면, AI는 ‘대화 상대’를 넘어 ‘우리 기관에 맞춰 일하는 동료’가 됩니다.

재정경제부 임직원 교육 자료 · Claude Code 입문 · 본 페이지의 통계·시뮬레이션은 학습용 예시입니다.
